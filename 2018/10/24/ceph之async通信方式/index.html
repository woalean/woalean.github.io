服务端总结

结构关系：

  1. 创建AsyncMessenger *msgr;
  2. 每个AsyncMessenger管理了:
    1）一个NetworkStack stack；
    2）一个Processor    列表；
    3）一个DispatchQueue派发队列；
  3. 每个NetworkStack管理了：
    1）一个workers队列
  4. 每个Processor组合了： AsyncMessenger, Worker, ServerSocket, EventCallback

启动逻辑：
-> 创建一个AsyncMessenger
-> 创建一个NetworkStack
-> 根据配置创建一个或多个Worker并出使化
-> 创建并初始化Worker的消息处理线程
-> 绑定socket
-> 根据配置创建一个或多个Processor并把对应的Worker，AsyncMessenger, EventCallback组合起来
-> 把一个（mds/mgr/osd/mon）dispatcher加入派发者队列
-> ready这个派发者的Messenger,对于AsyncMessenger而言，会进入accept状态
